# Toolkit - UI-фреймворк проекта

## Описание

`toolkit` — это единый, самодостаточный UI-фреймворк проекта Improve-ImgSLI. Он содержит все переиспользуемые UI-компоненты, менеджеры, диалоги, сервисы и ресурсы, необходимые для построения пользовательского интерфейса приложения.

## Структура

```
toolkit/
├── widgets/          # UI-компоненты
│   ├── atomic/    # Атомарные виджеты (кнопки, поля ввода, чекбоксы и т.д.)
│   ├── composite/ # Композитные виджеты (flyouts, диалоги, сложные компоненты)
│   ├── list_items/# Элементы списков
│   └── helpers/   # Вспомогательные утилиты для виджетов
├── managers/       # Менеджеры (ThemeManager, FontManager, IconManager)
├── services/       # Сервисы (IconService)
├── dialogs/        # Переиспользуемые диалоги
└── resources/      # Ресурсы тулкита
    ├── fonts/     # Шрифты
    └── styles/    # Стили (QSS) и темы
```

## Использование

### Импорт виджетов

```python
from toolkit.widgets.atomic import CustomButton, FluentCheckBox, BodyLabel
from toolkit.widgets.composite import BaseFlyout, Toast
```

### Импорт менеджеров

```python
from toolkit.managers import ThemeManager, FontManager
from toolkit.managers.icon_manager import get_icon_by_path
```

### Импорт сервисов

```python
from toolkit.services import get_icon_service, IconService
```

### Импорт диалогов

```python
from toolkit.dialogs import HelpDialog, BaseDialog
```

## Принципы организации

1. **Атомарные виджеты** (`widgets/atomic/`) — базовые, неделимые UI-компоненты, которые можно использовать независимо.
2. **Композитные виджеты** (`widgets/composite/`) — сложные компоненты, состоящие из нескольких атомарных виджетов.
3. **Менеджеры** — управляют глобальным состоянием UI (темы, шрифты, иконки).
4. **Сервисы** — предоставляют функциональность для работы с ресурсами (иконки, изображения).

## Разработка новых компонентов

При добавлении новых UI-компонентов:

1. Определите, является ли компонент атомарным или композитным.
2. Разместите его в соответствующем каталоге.
3. Обновите `__init__.py` для экспорта нового компонента.
4. Используйте менеджеры из `toolkit/managers` для обеспечения консистентности стилей.

## Зависимости

Toolkit зависит от:
- PyQt6
- Внутренних модулей проекта (`core`, `resources`)

Toolkit **не зависит** от специфичной логики приложения — он должен быть переиспользуемым и независимым.
