## Файлы и проекты

Загружайте изображения в списки сессии, вставляйте из буфера и открывайте или сохраняйте портативные проектные файлы, которые хранят раскладку сессий, настройки сравнения и копии исходных изображений.

### Загрузка изображений {#loading-images}

- **Добавить файлы** — кнопки добавления у стороны (или {{tr:menu.find_action}}) выбирают один или несколько файлов.
- **Перетаскивание** — в окно; укажите список или слот {{tr:workspace.session_types.multi_compare}}.
- **Вставка** (`Ctrl+V`) — из буфера; оверлей направления: стрелки или `WASD` выбирают сторону, `Esc` отменяет.

![Оверлей направления вставки]({{img:platform.file_project.paste_overlay}})
`Ctrl+V` — оверлей направления вставки.

### Списки {#lists}

- **{{tr:workspace.session_types.image_compare}}** — левый и правый списки независимы; менеджер списков — порядок, рейтинг, переименование, путь, свойства, удаление: [Списки и всплывающие панели](help://ui.lists_flyouts).
- **{{tr:workspace.session_types.multi_compare}}** — дроп по слотам вместо двух списков: [{{tr:workspace.session_types.multi_compare}}](help://multi_compare).

### Проекты {#projects}

- **Открыть / сохранить** — `.imgsli` из меню File (`Ctrl+Shift+O` / `Shift+S` / `Ctrl+Shift+S`, или {{tr:menu.find_action}}): **Сохранить** пишет в текущий файл (или открывает **Сохранить как**, если ещё не сохраняли); **Сохранить как** предлагает имя переименованной вкладки, иначе {{tr:menu.project_untitled}}. **Сохранить** также меняет имя файла, если вкладку переименовали. При открытии проекта активная вкладка получает имя файла. Восстанавливает сессии, настройки сравнения (split, diff, лупа и связанные фичи) и встроенные копии изображений.
- **Портативный пакет** — файл это ZIP: JSON сессий плюс папка `media/` с байтовыми копиями оригиналов (без перекодирования пикселей).
- **Отсутствующие исходники при сохранении** — если путь из списка больше не существует, проект всё равно сохраняется; это изображение пропускается, и показывается предупреждение.
- **Настройки приложения** — тема, язык и горячие клавиши остаются в настройках приложения и не входят в файл проекта.
